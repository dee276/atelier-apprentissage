String

Concaténation

"Hello" + " World"           # crée une nouvelle String

Symboles

Symbol 
:my_symbol                        # syntaxe
:test.object_id == :test.object_id #=> true  (unique en mémoire)
"test".object_id == "test".object_id #=> false (nouveaux objets)

:color.to_s   #=> "color"

Les symboles sont immuables et stockés une seule fois en mémoire. Préférer les symboles aux chaînes pour les clés de hash — c'est plus performant et idiomatique en Ruby.

04

Tableaux

Array

Création

arr = [1, 2, 3, 4, 5]

Array.new(5, true)          #=> [true, true, true, true, true]
Array.new(3) { |i| i * 2 } #=> [0, 2, 4]  ← préférer le bloc pour les tableaux imbriqués

Accès

arr[0]        #=> 1
arr[-1]       #=> 5   (dernier)
arr[1..3]      #=> [2, 3, 4]
arr.first(2)  #=> [1, 2]
arr.last(2)   #=> [4, 5]

Ajout / suppression

arr.push(6)   # ou arr << 6   — ajoute en fin
arr.pop       # retire le dernier
arr.pop(3)    # retire les 3 derniers
arr.unshift(0) # ajoute en tête
arr.shift     # retire le premier

Opérations ensemblistes

a = [1,2,3]; b = [2,3,4]
a + b               #=> [1,2,3,2,3,4]  concaténation
a - b               #=> [1]            différence
a & b               #=> [2,3]          intersection
a | b               #=> [1,2,3,4]      union
a.intersection(b)   #=> [2,3]          (identique à &)

Méthodes utiles

arr.sort           # tri croissant (nouveau tableau)
arr.uniq           # élimine les doublons
arr.uniq.sort     # chaînable
arr.flatten        # aplatit les sous-tableaux
arr.include?(3)   #=> true / false
arr.min / arr.max # valeur min/max
arr.length        # nombre d'éléments

05

Hashes

Hash

Création — deux syntaxes

# Rocket (ancienne)
old = { :color => "red", :number => 42 }

# Symbole-clé (moderne, préférée)
fav = { color: "red", number: 42 }

Lecture

fav[:color]               #=> "red"
fav[:missing]             #=> nil  (pas d'exception)
fav.fetch(:number)         #=> 42
fav.fetch(:missing, 42)   #=> 42   (valeur par défaut)
fav.fetch(:missing)        #=> KeyError !

Modification & introspection

fav[:movie] = "Inception"   # ajouter / modifier
fav.delete(:number)         # supprimer (retourne la valeur)
fav.merge(other)            # fusion non-destructive
fav.merge!(other)           # fusion destructive

fav.keys    #=> [:color, :movie]
fav.values  #=> ["red", "Inception"]
fav.key?(:color)  #=> true

06

Structures imbriquées

Nested arrays & hashes

Tableaux 2D — création et accès

# Toujours utiliser le bloc pour créer des tableaux 2D mutables
grid = Array.new(3) { Array.new(2) }   # ✓ objets indépendants
bad  = Array.new(3,  Array.new(2))   # ✗ 3 lignes = même objet !

mailboxes[0][0]      #=> "Adams"
mailboxes[-1][-2]    #=> "Smith"  (négatifs OK)
mailboxes.dig(1, 0)  #=> "Jones"  (sûr : nil si chemin absent)

Itération imbriquée

# Lignes seulement
mailboxes.each_with_index do |row, i|
  puts "Row:#{i} = #{row}"
end

# Lignes + colonnes
mailboxes.each_with_index do |row, r|
  row.each_with_index do |name, c|
    puts "[#{r}][#{c}] = #{name}"
  end
end

# Valeurs uniquement → flatten d'abord
mailboxes.flatten.each { |name| puts name }

Hashes imbriqués

vehicles = {
  alice: { year: 2019, make: "Toyota" },
  caleb: { year: 2020, make: "Honda"  }
}

vehicles[:alice][:year]     #=> 2019
vehicles.dig(:alice, :color) #=> nil (sûr)
vehicles[:zzz][:year]       #=> NoMethodError! (nil[:year])

# Filtrage
vehicles.select    { |k, v| v[:year] >= 2020 }         # → hash
vehicles.filter_map { |k, v| k if v[:year] >= 2020 }  # → [:caleb]
.dig est toujours plus sûr que l'enchaînement [][] pour les structures profondes — il retourne nil au lieu de lever une exception.

Boucles

Loop · While · Until · Times
loop / break
loop do
  i += 1
  break if i == 10
end
while
while i < 10 do
  i += 1
end
until (inverse de while)
until i >= 10 do
  i += 1
end
for…in (plages)
for i in 'a'..'d'
  puts i
end

Itérateurs numériques (idiomatique Ruby)

5.times { puts "hello" }              # répète 5 fois
10.times { |n| puts "iter #{n}" }     # index 0..9

5.upto(10)   { |n| print "#{n} " }   #=> 5 6 7 8 9 10
10.downto(5) { |n| print "#{n} " }   #=> 10 9 8 7 6 5

(1..5).each  { |n| puts n }             # plage inclusive  (1,2,3,4,5)
(1...5).each { |n| puts n }             # plage exclusive  (1,2,3,4)
En Ruby idiomatique, préférer les itérateurs (.each, .times) au for. Le for ne crée pas de nouveau scope — les variables de boucle restent accessibles après.

Énumérables

Enumerable
MéthodeRetourneUsage
.eachtableau originalEffets de bord (puts, log…)

.map / .collectnouveau tableauTransformation 1-pour-1

# each — itération simple
[1,2,3].each { |n| puts n }           # retourne [1,2,3]

# map — transformation
[-3, 2, -1].map { |n| n.abs }        #=> [3, 2, 1]

# select / reject
[1,2,3,4].select { |n| n.even? }    #=> [2, 4]
[1,2,3,4].reject { |n| n.even? }    #=> [1, 3]

# reduce — accumuler dans un hash
words = ["cat", "dog", "elephant"]
words.reduce({}) do |acc, word|
  acc[word] = word.length
  acc   # ⚠️ TOUJOURS retourner l'accumulateur
end
#=> {"cat"=>3, "dog"=>3, "elephant"=>8}

# reduce — somme (raccourci)
[1,2,3].reduce(:+)   #=> 6
Avec reduce, oublier de retourner acc à la fin du bloc retourne la dernière valeur assignée, pas l'accumulateur entier.

Prédicats de collection

include? · any? · all? · none? 
numbers = [5, 6, 7, 8]

numbers.include?(6)                   #=> true
numbers.include?(3)                   #=> false

numbers.any?  { |n| n > 5 }          #=> true  (au moins un)
numbers.any?  { |n| n < 5 }          #=> false

numbers.all?  { |n| n.is_a?(Integer) } #=> true  (tous)
numbers.all?  { |n| n.is_a?(String) }  #=> false

numbers.none? { |n| n.is_a?(String) }  #=> true  (aucun)
numbers.none? { |n| n > 10 }          #=> true
Mémo logique : any? = OU · all? = ET · none? = NON-OU. Tous court-circuitent dès que le résultat est déterminé.
number.integer? n'est pas disponible directement. Utiliser number.is_a?(Integer) ou number.is_a?(Numeric).

Méthodes

def · return · case

Retour implicite et paramètre par défaut

def lucky_number(number = 7)
  "Today's lucky number is #{number}"  # retour implicite
end
# return explicite seulement pour l'early exit

Early exit / return anticipé

def ascii_code(character)
  return 'Input Error' if character.size != 1
  character.ord
end
# ou avec ternaire : (character.size != 1) ? 'Input Error' : character.ord

case/when

def pet_pun(animal)
  case animal
  when 'cat' then puts 'Cats are purr-fect!'
  when 'dog' then puts 'Dogs are paw-some!'
  else           puts "I think #{animal}s have pet-tential!"
  end
  # puts retourne nil → la méthode retourne nil
end

Convention ? et !

def twenty_first_century?(year)
  year.between?(2001, 2100)  # retour bool implicite
end

# méthode?  → retourne true/false
# méthode!  → version destructive / lève exception

Blocs : {} vs do…end

# {} pour blocs d'une ligne
arr.map { |n| n * 2 }

# do...end pour blocs multi-lignes
arr.each do |n|
  result = n * 2
  puts result
end

Pièges courants

À retenir absolument

1 — Retour implicite (puts retourne nil)

def calculer(prix)
  total = prix + 5
  puts "terminé"  # ⚠️ retourne nil, pas total !
end

2 — Opérateur pelle << (mutation)

prenom = "Jean"
prenom + " Dupont"  #=> prenom inchangé ✓
prenom << " Dupont"  #=> prenom = "Jean Dupont" ⚠️

3 — Division entière

10 / 4        #=> 2   (tronqué)
10.to_f / 4  #=> 2.5 ✓

4 — Chaîne vs Symbole (objet_id)

"test".object_id == "test".object_id  #=> false (2 objets)
:test.object_id  == :test.object_id   #=> true  (1 objet)

5 — Valeurs falsy en Ruby

# SEULEMENT nil et false sont falsy — tout le reste est truthy
if 0   # vrai  (≠ JavaScript)
if ""  # vrai  (≠ Python)
if []  # vrai  (≠ Python)

6 — map vs each

arr = [1, 2, 3]
arr.each { |n| n * 2 }   #=> [1, 2, 3]  ← retourne l'ORIGINAL
arr.map  { |n| n * 2 }   #=> [2, 4, 6]  ← nouveau tableau

7 — Plages .. vs ...

(1..5).to_a   #=> [1, 2, 3, 4, 5]  (inclusive)
(1...5).to_a  #=> [1, 2, 3, 4]     (exclusive)

8 — Array.new partagé (tableaux 2D)

bad  = Array.new(3,  Array.new(2))  # ⚠️ même objet × 3
good = Array.new(3) { Array.new(2) } # ✓ objets distincts

Quiz

Auto-évaluation

10 questions
Score final : / 10
1. Que retourne 10 / 4 en Ruby ?
✓ Division entière : Ruby tronque le résultat. Utiliser 10.to_f / 4 pour obtenir 2.5.
✗ Ruby effectue une division entière. 10 / 4 #=> 2.
2. Quelle méthode retourne un nouveau tableau transformé ?
✓ .map (ou .collect) retourne un nouveau tableau. .each retourne le tableau d'origine.
✗ .map retourne un nouveau tableau. .each retourne l'original.
3. Que vaut :test.object_id == :test.object_id ?
✓ Les symboles sont uniques en mémoire — le même symbole a toujours le même object_id.
✗ Les symboles sont immuables et stockés une seule fois : même object_id, donc true.
4. Que retourne puts "hello" ?
✓ puts affiche la valeur mais retourne toujours nil — piège classique avec le retour implicite.
✗ puts retourne toujours nil, quelle que soit la valeur affichée.
5. Comment accéder en sécurité à un hash imbriqué sans risque de NoMethodError ?
✓ .dig retourne nil si un chemin est absent, au lieu de lever une exception.
✗ .dig est la méthode sûre : hash[:a][:b] lève NoMethodError si :a retourne nil.
6. Quelle valeur est truthy en Ruby ?
✓ En Ruby, seuls nil et false sont falsy. Le 0, "", et [] sont truthy !
✗ En Ruby, seuls nil et false sont falsy — 0, "", et [] sont truthy (≠ JS/Python).
7. Que retourne (1...5).to_a ?
✓ ... est une plage exclusive : elle exclut la borne supérieure.
✗ ... exclut la borne supérieure. (1..5) inclurait 5.
8. Avec reduce, que doit obligatoirement retourner le bloc à chaque itération ?
✓ Le bloc doit retourner l'accumulateur (acc) sinon la prochaine itération reçoit la dernière valeur assignée.
✗ Le bloc doit retourner l'accumulateur à chaque tour — sinon reduce perd le résultat accumulé.
9. Quelle est la différence entre Array.new(3, Array.new(2)) et Array.new(3) { Array.new(2) } ?
✓ Sans bloc, les 3 lignes pointent vers le même tableau — modifier l'une modifie toutes les autres.
✗ Sans bloc, Ruby réutilise le même objet pour les 3 éléments. Le bloc crée des objets indépendants.
10. Quelle méthode combine .map et .compact en une seule passe ?
✓ .filter_map (Ruby 2.7+) itère, transforme, et élimine les nil en une seule passe efficace.
✗ .filter_map (Ruby 2.7+) combine map + compact en une seule itération.
